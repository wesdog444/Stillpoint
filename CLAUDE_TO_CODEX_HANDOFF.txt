WHAT CHANGED

Browser / Social mini-app shell
- src/screens/BrowserScreen.tsx:
  - Removed "Block these", "0 ads", and "0 suggested" from the top chrome.
  - Top chrome now shows only a timer.
  - Timer is computed from elapsed wall-clock time via a start timestamp, not a blind increment.
  - Added persistent login props:
    - sharedCookiesEnabled
    - domStorageEnabled
    - thirdPartyCookiesEnabled
    - incognito={false}
  - Added a destination rail for each social platform.
  - Instagram destinations intentionally exclude Reels.
  - Toolbar behavior now matches the user's request:
    - Back calls WebView goBack.
    - Home returns to the Stillpoint Social screen inside the app.
    - Refresh calls WebView reload.
    - Forward calls WebView goForward.
    - Account opens an Account Manager overlay.
  - Account Manager clarifies that logins are remembered by the site inside the persistent WebView and offers account/profile destinations.

Social destination model
- src/social/destinations.ts:
  - Adds platform-specific destinations for Instagram, YouTube, X, TikTok, Facebook, Snapchat.
  - Keeps Breathe as a safe destination.
  - Instagram destinations are Messages, Search, Profile, Account, Breathe; no Reels.
- src/social/__tests__/destinations.test.ts:
  - Verifies Instagram does not expose Reels.

Sanitizer
- src/sanitizer/rules.ts:
  - Strengthened Instagram selectors:
    - a[href*="/reels/"]
    - a[href*="/explore/"]
    - svg[aria-label="Reels"]
    - svg[aria-label="Explore"]
    - div[aria-label*="Suggested"]
    - main[role="main"] article
    - section main article
- src/sanitizer/injection.ts:
  - Adds cleanStillpointSurfaces().
  - Runs cleanup immediately and repeats with setInterval so dynamically inserted social surfaces are hidden after load.

Blocks tab
- src/screens/BlocksScreen.tsx:
  - Replaces placeholder with Redirect Scheduler / Control Center.
  - Shows active hours examples.
  - Shows stillpoint://breathe.

Profile tab

IMPORTANT LIMITATIONS / REALITY CHECKS
- Stillpoint cannot programmatically create, enable, or disable the user's personal Shortcuts automations. The Shortcuts app exposes App Intents/App Shortcuts, but personal automation toggles stay under user control.
- The social feature still uses WebView underneath. That is expected. The goal is a Stillpoint-owned mini-app shell and strong sanitization, not a native Instagram clone.
- Instagram/YouTube/X/TikTok/Facebook/Snapchat selectors will drift over time. Reels hiding is stronger now, but real-device testing is still required.
- Login persistence is configured, but the user should verify on device by logging into Instagram once, closing/reopening Stillpoint, and reopening Instagram.
