=== V1.1 FEEDBACK TO BUILD ===

(1) ONBOARDING — make it more enticing and interactive.
    Current state: 4 plain dark text-only steps (Welcome / How Stillpoint works /
    Gentle reminders / Set up redirect Shortcuts). Functional but flat.
    Wanted: livelier, more engaging onboarding — imagery/illustration, motion/
    animation, and interactivity (e.g. let the user pick which social sites they
    care about DURING onboarding, preview the orb, etc.). Brainstorm the design.
    Keep the Personal Edition copy honest; keep the "never says block" guard.

(2) BREATHE DEEP LINK — bug: stillpoint://breathe goes nowhere.
    Plan 7 described a "breathe" landing target for redirect automations on apps
    with no sanitized version, but it was never built. src/nav/linking.ts has NO
    `breathe` route and there is no Breathe screen.
    Wanted: build a real Breathe screen (a calm breathing-exercise interstitial)
    and add a `breathe` route to the linking config so stillpoint://breathe works.

(3) SCREEN TIME PERMISSIONS — DO NOT BUILD. Permanently out of scope.
    The owner asked the app to request Screen Time permissions. This is the iOS
    Family Controls / Screen Time API and requires the Apple entitlement
    com.apple.developer.family-controls, granted only to reviewed PAID developer
    accounts. A Sideloadly-signed build cannot have it — adding the code would
    crash the app / fail install. This is the entire reason Stillpoint is
    "Personal Edition" (see spec section 2). Do NOT attempt it. The sideload-
    compatible substitute already exists: the stillpoint:// redirect Shortcuts.
    If Codex wants to help here, the only honest move is to (politely, in the
    build summary) restate this constraint — not implement it.

(4) SOCIAL TAB — make it a fully integrated "sanitized app", not a raw browser.
    Reference app: "SocialLite" (the owner has screenshots; description below).
    IMPORTANT: SocialLite is itself a WebView wrapper — you cannot natively
    re-implement Instagram/etc. What makes it feel integrated is the custom chrome
    AROUND the webview. Build that polish on top of the existing BrowserScreen:
      - A site list styled like SocialLite: rounded cards, each with the site's
        real icon, a per-site settings gear, and a chevron. Tagline under the
        app name. (SocialLite also lists Facebook & Snapchat — consider adding
        them to the current 4: Instagram, X, YouTube, TikTok.)
      - A branded transition/loading screen when opening a site ("Applying your
        preferences — Setting up content filters for <Site>") instead of a blank
        webview load.
      - Custom chrome over the webview: a top stats strip (time in session,
        ads blocked, suggested/feed items removed) with a "Block these" style
        action, and a floating control toolbar (back / home / refresh / forward /
        settings) — like SocialLite's.
      - Heavier, more thorough sanitization so the page genuinely feels calm.
    The owner's words: "fully integrate the website and then Stillpoint transform
    into the sanitized app." Goal = it should not feel like Safari popped up.
