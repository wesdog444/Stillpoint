Stillpoint handoff - Instagram fullscreen/Reels bug fix
Date: 2026-05-21
Current branch: codex/social-fullscreen-reels-fix

SESSION SUMMARY
- User provided a phone screenshot showing that:
  - Stillpoint destination chips were visible above Instagram.
  - Stillpoint bottom tab bar was visible below Instagram.
  - Instagram's Reels icon remained in the middle of Instagram's bottom nav.
  - The actual site should feel like only the purple-box area is visible.
- I treated this as a concrete bug pass.

WHAT CHANGED
- BrowserScreen:
  - Removed the internal Stillpoint destination rail/chips from the site browser.
  - WebView now receives the sanitizer through both:
    - injectedJavaScriptBeforeContentLoaded
    - injectedJavaScript
  - This gives the sanitizer a chance to run earlier, before Instagram fully paints.
- RootNavigator:
  - Hides the bottom Stillpoint tab bar when Social stack route is Browser.
  - Restores it on SocialHome and other tabs.
- Instagram sanitizer:
  - Added `nav a[href*="/reels/"]`.
  - Added `hideInstagramReelsNav()` inside the injected script.
  - The function targets `svg[aria-label*="Reels"]`, `a[href*="/reels"]`, and `a[href*="/reel"]`, then hides the closest link/tab/nav/card container.
