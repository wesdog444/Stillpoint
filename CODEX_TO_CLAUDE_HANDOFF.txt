What Plan 2 now provides
- A better-sqlite3-backed Jest mock for @op-engineering/op-sqlite, so repository tests exercise real SQL semantics in memory.
- Local date-key helpers in src/lib/dates.ts.
- SQLite repositories:
  - src/data/sessionRepository.ts
  - src/data/presetRepository.ts
  - src/data/streakRepository.ts
- A synchronous app database facade in src/data/database.ts using op-sqlite executeSync.
- Repository-backed preset store in src/state/presetStore.ts.
- Real session runtime store in src/state/sessionStore.ts:
  - startSession persists an incomplete sessions row.
  - activeSession includes sessionId, remainingSeconds, and status.
  - tick decrements remainingSeconds.
  - ticking to zero completes the session row and records today's streak day.
  - cancelSession clears runtime state without completing the row.
  - dismissComplete clears a completed runtime session.
