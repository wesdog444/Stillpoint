# CI Build & Sideload Implementation Plan

> **For agentic workers:** This is an infrastructure plan, not a TDD plan. Tasks create config/workflow/docs files and run git/CI operations. There is no Jest to run for most tasks — verification is "the file is well-formed" and ultimately "the GitHub Actions build succeeds and produces an installable .ipa". Use superpowers:executing-plans or subagent-driven-development to work through it task-by-task.

**Goal:** Produce an installable Stillpoint `.ipa` that the project owner can sideload onto a personal iPhone with Sideloadly — built in the cloud on a GitHub Actions macOS runner, since the owner develops on Windows.

**Architecture:** The repo gets a GitHub remote. A GitHub Actions workflow runs on a `macos` runner: it checks out the code, runs `expo prebuild` to generate the native iOS project (Continuous Native Generation — `ios/` is gitignored and regenerated), builds an **unsigned** `.app` with `xcodebuild`, repackages it into an unsigned `.ipa`, and uploads it as a build artifact. The owner downloads that artifact and signs+installs it with Sideloadly using their own Apple ID. A `SIDELOAD.md` documents the download/sign/install steps and the one-time Shortcuts redirect-automation setup.

**Tech Stack:** GitHub Actions (`macos` runner), Node 22, Expo SDK 54 prebuild, `xcodebuild`, CocoaPods, Sideloadly (owner's machine).

**Prerequisite reading:**
- `docs/superpowers/specs/2026-05-19-stillpoint-design.md` section 7.2 (build & distribution) and section 2 (Personal Edition / 7-day re-sign reality).
- `app.json` — already declares `name: "Stillpoint"`, `slug: "stillpoint"`, `scheme: "stillpoint"`, `ios.bundleIdentifier: "com.wesle.stillpoint"`.
- `.gitignore` — already ignores `/ios` and `/android` (native dirs are regenerated by prebuild).

**Scope note — App Intents deferred:** The roadmap mentioned Swift App Intents (so the iOS Shortcuts app can call *actions into* Stillpoint). They are intentionally NOT in this plan: Plan 7 already shipped `stillpoint://` deep linking, which is everything the Shortcuts *redirect* automations need (an automation that does "Open App → Stillpoint" / opens `stillpoint://sanitized/instagram`). App Intents only add the fancier "Shortcuts action: Start a Stillpoint focus session" — a refinement, not required for v1, and not verifiable without a Mac. They can be a later optional Plan 9.

## Task 3: Add the GitHub Actions build workflow

A workflow that builds an unsigned `.ipa` on a macOS runner. It runs on manual dispatch and on pushes to `master`.

**Files:**
- Create: `.github/workflows/build-ipa.yml`

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/build-ipa.yml`:
```yaml
name: Build unsigned IPA

on:
  workflow_dispatch:
  push:
    branches: [master]

jobs:
  build:
    runs-on: macos-15
    timeout-minutes: 45

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Expo prebuild (generate native iOS project)
        run: npx expo prebuild --platform ios --no-install
        env:
          CI: 1

      - name: Install CocoaPods
        working-directory: ios
        run: pod install

      - name: Build unsigned .app
        working-directory: ios
        run: |
          xcodebuild \
            -workspace Stillpoint.xcworkspace \
            -scheme Stillpoint \
            -configuration Release \
            -sdk iphoneos \
            -derivedDataPath build \
            CODE_SIGNING_ALLOWED=NO \
            CODE_SIGNING_REQUIRED=NO \
            CODE_SIGN_IDENTITY="" \
            build

      - name: Package .app into an unsigned .ipa
        working-directory: ios
        run: |
          APP_PATH="build/Build/Products/Release-iphoneos/Stillpoint.app"
          if [ ! -d "$APP_PATH" ]; then
            echo "Build product not found at $APP_PATH" >&2
            ls -R build/Build/Products || true
            exit 1
          fi
          rm -rf Payload Stillpoint.ipa
          mkdir Payload
          cp -R "$APP_PATH" Payload/
          zip -r -y Stillpoint.ipa Payload >/dev/null
          echo "Created $(pwd)/Stillpoint.ipa"

      - name: Upload IPA artifact
        uses: actions/upload-artifact@v4
        with:
          name: Stillpoint-unsigned-ipa
          path: ios/Stillpoint.ipa
          if-no-files-found: error
```

- [ ] **Step 2: Validate the YAML locally**

Run (from the repo root):

## Task 4: Write the sideload guide

**Files:**
- Create: `docs/SIDELOAD.md`

- [ ] **Step 1: Create docs/SIDELOAD.md**

Create `docs/SIDELOAD.md`:
```markdown
# Installing Stillpoint on your iPhone

Stillpoint is a Personal Edition app — it is not on the App Store. You build it
in the cloud and install it yourself with Sideloadly.

## 1. Build the IPA

1. Push to `master` (or open the repo on GitHub → **Actions** →
   **Build unsigned IPA** → **Run workflow**).
2. Wait for the macOS build to finish (~15-30 min).
3. Open the finished run → **Artifacts** → download **Stillpoint-unsigned-ipa**.
4. Unzip it — inside is `Stillpoint.ipa`.

## 2. Sign and install with Sideloadly

1. Install Sideloadly on your Windows PC (https://sideloadly.io) and iTunes +
   iCloud (the non-Microsoft-Store versions) so Sideloadly can see your device.
2. Connect your iPhone by USB and trust the computer.
3. Open Sideloadly, drag in `Stillpoint.ipa`, enter your Apple ID, and click
   **Start**. Sideloadly re-signs the app with your Apple ID and installs it.
4. On the iPhone: **Settings → General → VPN & Device Management** → trust your
   Apple ID's developer profile.
5. Launch Stillpoint.

### The 7-day reality

A free Apple ID signs apps for **7 days**. After that Stillpoint stops
launching — reconnect the phone and re-run Sideloadly to re-sign. A paid Apple
Developer account ($99/yr) extends this to 1 year.

## 3. Set up redirect Shortcuts (optional but recommended)

Stillpoint's in-app onboarding walks you through this. In short, for each social
app you want to redirect: open the **Shortcuts** app → **Automation** → **+** →
**App** → choose the app → **Is Opened** + **Run Immediately** → action
**Open App → Stillpoint**. Now every reflexive tap on that app bounces you into
Stillpoint's calm, sanitized version instead.

## What this build can and cannot do

- **Can:** sanitized in-app browser, focus sessions, friction, streaks, stats,
  the `stillpoint://` deep links the redirect automations use.
- **Cannot:** OS-enforced blocking of the native apps — that needs Apple's
  Family Controls entitlement, which is not available to a sideloaded build.
  See the design spec, section 2.
```

```bash

```

## Task 5: Run the build and retrieve the IPA

This task is owner-driven and happens on GitHub, not locally.

**Files:** none.

- [ ] **Step 1: Push the new commits**

```bash
git push
```
The push to `master` triggers the **Build unsigned IPA** workflow automatically.

- [ ] **Step 2: Watch the build**

On github.com → the repo → **Actions** tab → the running **Build unsigned IPA**
workflow. Wait for it to finish.

- [ ] **Step 3: Handle a build failure (if any)**

If the build fails, the most likely causes and fixes:
- **Scheme name mismatch** — if `xcodebuild` reports the `Stillpoint` scheme is
  not found, run `npx expo prebuild --platform ios` locally-equivalent logic is
  not possible on Windows; instead read the failing log for the actual scheme
  name under `ios/` and update both the `-scheme` and `-workspace` values in
  `.github/workflows/build-ipa.yml`, then commit and push again.
- **Build product path** — if the "Package" step reports the `.app` was not
  found, read the `ls -R build/Build/Products` output the step prints and adjust
  `APP_PATH` accordingly.
- **CocoaPods / prebuild errors** — read the log; these are usually a dependency
  needing a newer pod; re-running the workflow often resolves transient network
  failures.
Iterate on the workflow file until the run is green.

- [ ] **Step 4: Download and verify**

Download the **Stillpoint-unsigned-ipa** artifact, unzip, confirm `Stillpoint.ipa`
exists and is non-trivial in size (tens of MB). Then follow `docs/SIDELOAD.md`.

There is no commit for this task unless Step 3 required a workflow fix.

- The repo has a GitHub `origin` remote and `master` is pushed.
- `.github/workflows/build-ipa.yml` exists, is valid YAML, and runs on push/dispatch.
- The workflow completes green on a macOS runner and uploads a `Stillpoint.ipa` artifact.
- `docs/SIDELOAD.md` documents the full download → Sideloadly → install flow.
- The owner can install the artifact on a personal iPhone via Sideloadly.

At that point Stillpoint is feature-complete for Personal Edition v1 and installable.
